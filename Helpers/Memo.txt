RESHARPER - GENERATE - EQUALITY MEMBERS (/COMPARER)

Nagyon hasznos, sokat gyorsít; legenerálja az Equals és a GetHashCode fv-ek felüldefiniáltját. 
Van viszont egy kis hibája: Az Equals-ba nem teszi bele a GetHashCode-ot. Kézzel kell javítani. Pl:
		public bool Equals(ExpenseItem other)
		{
			if (ReferenceEquals(null, other))
				return false;
			if (ReferenceEquals(this, other))
				return true;
--> 		if (this.GetHashCode() != other.GetHashCode())
-->			return false;
			return base.Equals(other) && _category == other._category;
		}
Tehát a 2 másolandó sor mindig ez:
if (this.GetHashCode() != other.GetHashCode())
return false;

Equality Comparerhez pedig ez:
if(GetHashCode(x) != GetHashCode(y))
return false;

Ehhez megjegyzés, hogy
- A Resharperes Equality Comparer generálás azért nem épp tökéletes. Nem foglalkozik az õsosztállyal...
	(érdemes legenerálni az IEquatable-t, és onnan leutánozni az Equals és a GetHashCode metódusokat, mert
	 azon a részen figyelembe veszi az õsosztályt is)
- Viszont szükség van rá a Xaml UI modelleknél, mert ha implementáljuk bennük az IEquatable-t, akkor nagyon
	magic hibák lesznek a felületen... (ez az infó 4-5 óra szenvedés eredménye)
